# Миграция на WSL2

## Преимущества WSL2 для HTX проекта

1. **Улучшенная производительность**:
   - Полноценная виртуализация Linux с лучшей производительностью файловой системы
   - Более быстрый доступ к файлам в Linux-файловой системе

2. **Лучшая совместимость с ML-библиотеками**:
   - scikit-learn, pandas, numpy работают стабильнее
   - Отсутствие проблем с путями и зависимостями

3. **Поддержка GPU для ML-моделей**:
   - WSL2 поддерживает GPU для тяжелых ML-задач
   - Возможность использования CUDA для оптимизации

4. **Улучшенная совместимость с Docker**:
   - Docker в WSL2 работает нативно
   - Упрощается деплой и тестирование контейнеризированных приложений

## Как использовать проект в WSL2

1. **Запуск проекта**:
   ```bash
   # Из Windows PowerShell:
   wsl -d Ubuntu -- bash -c "cd ~ && ./htx_project/start_wsl2.sh"
   
   # Или из WSL2 напрямую:
   cd ~/htx_project && ./start_wsl2.sh
   ```

2. **Доступ к сервисам**:
   - Backend: http://localhost:8004
   - Frontend: http://localhost:3000
   - API Docs: http://localhost:8004/docs

3. **Работа с файлами**:
   - Храните рабочие файлы в WSL2 файловой системе (~/)
   - Используйте `/mnt/c/` или `/mnt/e/` только для обмена с Windows

## Решенные проблемы

1. **Проблемы с ML-зависимостями** (numpy, scikit-learn)
2. **Медленный доступ к файлам** в Windows из WSL1
3. **Конфликты путей** между Windows и Linux

## Развертывание в production

Для production-окружения рекомендуется использовать Docker контейнеры, собранные в WSL2:

```bash
cd ~/htx_project
docker-compose up -d
```

## Инструкции по переносу проекта

### 1. Подготовка WSL2

```bash
# Проверка текущей версии WSL
wsl --list --verbose

# Обновление до WSL2 (если нужно)
wsl --set-default-version 2
wsl --set-version Ubuntu 2
```

### 2. Подготовка окружения в WSL2

```bash
# Создание директории проекта
mkdir -p ~/htx_project

# Копирование файлов из Windows
cp -r /mnt/e/Htx_project_attemp_101/* ~/htx_project/
cd ~/htx_project

# Создание виртуального окружения в WSL2
python -m venv .venv_wsl2
source .venv_wsl2/bin/activate

# Установка зависимостей
pip install -r requirements.txt
pip install scikit-learn pandas redis aiohttp
```

### 3. Первый запуск

```bash
# Делаем скрипт исполняемым
chmod +x start_wsl2.sh

# Запуск
./start_wsl2.sh
```

## Отладка и возможные проблемы

1. **Проблема**: Порт занят другим процессом
   **Решение**: `sudo lsof -i:8004` для поиска процесса, `kill -9 PID` для завершения

2. **Проблема**: Недостаточно прав для установки пакетов
   **Решение**: Использование виртуального окружения или `sudo pip install`

3. **Проблема**: Ошибки импорта модулей
   **Решение**: Убедиться в правильной активации виртуального окружения WSL2

4. **Проблема**: Проблемы с доступом к базе данных
   **Решение**: Проверить пути к файлам БД, использовать абсолютные пути в конфигурации
